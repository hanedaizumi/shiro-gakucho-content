# AI システム開発｜技術要件

## 固定スタック（変更不可）

| レイヤー | 技術 |
|---|---|
| フロント / バック | **Next.js（App Router）** |
| ORM | **Prisma** |
| スタイリング | **Tailwind CSS** |
| DB | **PostgreSQL** |
| 本番環境 | **Google Cloud** |

---

## 開発方針

### Next.js（App Router）
- `app/` ディレクトリ構成を使用する（Pages Router は使わない）
- Server Components をデフォルトとし、Client Components は `'use client'` が必要な箇所のみ
- API は Route Handlers（`app/api/**/route.ts`）または Server Actions を使用

### Prisma
- スキーマは `prisma/schema.prisma` で一元管理
- マイグレーションは `prisma migrate dev` / 本番は `prisma migrate deploy`
- DB アクセスは Prisma Client 経由（生 SQL は必要最小限）

### Tailwind CSS
- ユーティリティクラスを基本とする
- 共通 UI は `@/components` にコンポーネント化

### PostgreSQL
- ローカル開発：Docker または Cloud SQL Auth Proxy 経由
- 接続文字列は環境変数 `DATABASE_URL` で管理（`.env` は Git 管理外）

---

## 本番環境（Google Cloud 想定）

| 用途 | 推奨 GCP サービス |
|---|---|
| Next.js アプリ | **Cloud Run** |
| PostgreSQL | **Cloud SQL（PostgreSQL）** |
| シークレット | **Secret Manager** |
| 静的アセット | Cloud Storage + CDN（必要時） |
| CI/CD | Cloud Build または GitHub Actions → Cloud Run デプロイ |

---

## 環境変数（最低限）

```
DATABASE_URL=
NEXTAUTH_SECRET=   # 認証を使う場合
NEXTAUTH_URL=      # 認証を使う場合
```

---

## 禁止事項

- Pages Router の新規採用
- Prisma 以外の ORM 新規導入（TypeORM / Drizzle 等）
- 本番 DB を SQLite / MySQL に変更
- AWS / Vercel 等を本番の主軸とする構成（GCP 前提）
