## 執筆前に必ず確認する「やりがちミス」一覧

台本103のレビューで発見した問題点を含む、過去の全指摘を執筆視点で抽象化。台本を書き始める前と書き終えた後の両方で確認すること。

## 【導入セクション】

### ✅ 1. 冒頭1文目は断言+身近感セット
- 断言になっているか（「〜かもしれません」「〜可能性が」は禁止）
- **30〜50代男性の日常（楽天・スーパー・給料・NISA）に即座につながるか**
- 「財務大臣が〜」だけでは自分ごとにならない。「→だからあなたの○○が変わる」の文脈まで1〜2行で示す

### ✅ 2. 自己紹介は3文固定
- 権威1文（「年間10億円を運用するシロです」）
- 失敗1文（「2018年に500万円が200万円になった」）
- 信頼1文（「だからこそ今日の話には自信があります」）
- 「Web3事業100個」「稼ぎながら」は禁止

### ✅ 3. 疑問提起は一人称転換型
- 禁止：「こんな風に思っていませんか？当然の反応です。」
- 正解：「〜と思いましたよね。僕も最初そう思ってました。でも〇〇で変わりました。」

### ✅ 4. 「そう思いますよね」系は台本全体で3回以内
- 執筆後に全文検索して数える
- 超えている場合は「その通りです」「聞いてください」「違います」等に変換

### ✅ 5. 導入〜大目次②の間に必ず1回「視聴者の生活」に触れる
- 約4,000字の間、NISAや給料や物価に一切触れないのは❌
- 「あなたの○○に直接関係する話です」という橋渡し文を1箇所は入れる

## 【本編（大目次①②）セクション】

### ✅ 6. 600字以上続く説明にはフックを必ず1つ入れる
- 「でもここがヤバいんですが──」
- 「○○って聞いたことありますか？」
- 「ここが今日一番○○に関係ある話です。聞いてください。」
- 法律や制度の解説が続く箇所は特に要注意

### ✅ 7. クイズ形式を最低1箇所作る
- 大きな数字の直前に「いくらになると思いますか？」
- データの直前に「○○って知ってますか？」
- クイズが視聴維持率の「山」を作る

### ✅ 8. 並列比較フォーマットを活用する
- 「○○ vs ○○」を比較できる場面で表形式または対比形式にする
- 比較機会の例：XRP vs SWIFT、NISA vs 貯金 vs XRP+トレード、自己流 vs インジケーター活用
- テロップとして映えるかで判断する

### ✅ 9. 専門用語は必ず中学生レベルに翻訳する
- SEC・CFTC → 「アメリカの金融規制当局（日本でいう金融庁みたいな組織）」
- 法律名 → 「○○みたいな法律」で解説
- 翻訳の際は**直前の比喩と連動**させると流れが自然になる

### ✅ 10. 比喩は「より強い印象を与えるもの」を選ぶ
- 基準：「伝書鳩」「消防車のホース」「おにぎり10個が3万円」レベルの映像的インパクト
- 「○○年なのに△△」という時代との対比を入れると納得感が高まる
- 弱い比喩は積極的に差し替える

### ✅ 11. ニュース列挙の締めは「あなたはまだ持ってないのに。」
- 「国も銀行も全員動いている。あなただけ動いていない」という構造を使う
- これが行動喚起に直結する最強フレーズ

## 【大目次③（トレード誘導）セクション】

### ✅ 12. 消去法でトレードに誘導する（押し売り禁止）
「元手を増やす方法は3つしかない」
→ ①副業・転職（1〜2年かかる）
→ ②節約（月3万が限界）
→ ③トレード（今日から）
→ 「答えは明らかですよね」

この消去法を経由せずに「XRP+トレードが答えです」と結論を出すと、視聴者に「売り込まれた」感が残る。

### ✅ 13. 失敗談は必ず損失額を先出しする
- 禁止：「2018年に500万円入れました。直後から暴落して200万円になりました。」（説明が先）
- 正解：「300万円消えました。2018年に。」（パンチライン先出し）
- 数字を最初に出すことで「なぜ？」の好奇心が次へ引っ張る

### ✅ 14. ③変換の知覚を入れる（「以前→きっかけ→今」）
- シロ自身の変化プロセスを見せる
- 「以前は〜だった」「ある出来事がきっかけで変わった」「今は〜できるようになった」の三段構成
- 「完成したシロ」を見せるだけでなく「変化の軌跡」を見せることで視聴者が「自分も変われそう」と感じる

### ✅ 15. インジケーターは「魔法ではない」を先出しする
- 「入れたら自動で稼げる魔法のツールではありません」を必ず先に言う
- 補助輪の比喩：「補助輪は勝手に走ってくれない。でも転ばないための支えになる」
- 「ペダルを漕ぐのはあなた自身」は説教臭い→「そこを目指してほしいんです」に変換

## 【CTA・エンディングセクション】

### ✅ 16. メルマガ特典は全列挙しない
- 最強2つを「正直有料級です。受け取らないと損」と先出し
- 「全部で5つあります」とまとめる
- 禁止：5つ全部をベタ書き

### ✅ 17. 逆説フィルタリングを入れる
- 「何も勉強したくない人・ツールで楽して稼ぎたい人は登録しないでください」
- これで登録率が上がる（本気の人だけ来る演出）

### ✅ 18. Discordコミュニティへの誘導を1〜2行入れる
- メルマガCTAの直後か大目次③の適切な位置に
- 「シロが運営する完全無料のDiscordコミュニティでは〜を毎日公開しています」

### ✅ 19. エンディングはYes/No型コメント促進
- 禁止：「今日の学びをアウトプットしましょう」（抽象的でコメントしにくい）
- 正解：「あなたは今XRPを持っていますか？持ってる人は枚数、まだの人は『まだ』って書いてください」
- 「どちらが多いか、一緒に見てみましょう」で能動的コメントを誘う

---

## 【まとめセクション】

### ✅ 20. 「30秒でまとめます」宣言型で始める
- 「今日の内容を振り返ります」ではなく「最後に、今日の話を30秒でまとめます」
- 時間を宣言することで最後まで聞くハードルが下がる

### ✅ 21. インターネット比喩には数字を乗せる
- 「1990年代のインターネットが今のXRP」は使い古されている
- 数字を追加：「1990年にインターネット株に10万円入れた人が今いくらになったか知っていますか？○億円です」

## 【全体チェック（執筆後の見直し用）】

| チェック項目 | 確認方法 |
|---|---|
| 「そう思いますよね」系が3回以内か | 全文検索してカウント |
| 「ということで」が3回以内か | 全文検索してカウント |
| 「〜について解説します！」がないか | 全文検索 |
| 冒頭1文目が断言で身近感があるか | 声に出して読む |
| 600字以上フックなしの区間がないか | 各段落の文字数を目算 |
| クイズが1箇所以上あるか | 目視確認 |
| 並列比較が1箇所以上あるか | 目視確認 |
| 失敗談の数字が先出しになっているか | 失敗談箇所を確認 |

| エンディングにYes/No型コメント促進があるか | エンディングを確認 |
| Discordへの誘導があるか | 全体を目視確認 |
| ショート切り抜き候補が5箇所以上あるか | 「映像として30秒で成立するか」で確認 |
