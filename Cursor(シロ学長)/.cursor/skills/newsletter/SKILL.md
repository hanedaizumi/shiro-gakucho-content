# メルマガ作成スキル

## ベーススキルの読み込み（最初に実行）

1. `shiro-master` スキルを読み込む → 「誰に・何のために・どう届けるか」の軸を確認
2. `goal-seek` スキルの「4. メルマガ」セクションを確認 → 構成型（共感→自己開示→知識→行動→締め）を把握

## 事前読み込み（必須）

### ① 定義・ルール
- `01_定義フォルダ/03_メルマガ作成ルール.md`（4つの型・構成ルール・ワークフロー）
- `01_定義フォルダ/00_共通定義（ペルソナ・理念・トーン）.md`（ペルソナ・哲学・トーン）

### ② アーカイブ（必須）
- `03_アーカイブ/03_メルマガ_アーカイブ/`（過去メルマガ — 被り防止・トーン学習）

### ③ リファレンス（必須 — 全ファイル）
- `04_リファレンス/episode.md`（エピソード一次情報）
- `04_リファレンス/trade_principles.md`（トレード哲学）
- `04_リファレンス/weakness.md`
- `04_リファレンス/5th_grade_level_samples.md`

### ④ インプット（必須）
- `07_インプット/メルマガ用/★メルマガトレース元.md`（型・トレース元文章・テーマ・一次情報・CTA先）

## メルマガ作成の方針

- **型を必ず確認する**：インプットファイルのどの型セクションが記入されているか確認し、その型のルールに従って生成する
- **トレース元の構造・フック・流れ**を分析し、シロ学長の一次情報（`episode.md` 等）で中身を置き換える
- **過去メルマガと被らない**テーマ・表現にする（`03_メルマガ_アーカイブ/` で確認）
- **インプットに書かれた情報のみ**を使用（テーマが不明な場合は確認してから作業する）

---

## 4つの型と配信バランス

| 型 | 目的 | 8通中の配分 |
|---|---|---|
| ① 課題解決型 | 読者の悩みを一次体験で解決 → 「やってみよう」 | 3通 |
| ② 独自の切り口型 | 世の中の出来事を自分フィルターで再解釈 → 「そういう見方か」 | 2通 |
| ③ 価値観・マインド型 | 信念・価値観の変化をストーリーで語る → 「この人の考え方好きだ」 | 2通 |
| ④ リアルタイム共有型 | 今起きていることをそのまま届ける → 「一緒に歩んでいる感覚」 | 1通 |

## クイックスタート

1. `★メルマガトレース元.md` の記入済みセクションを確認し、型を特定する
2. **件名を3〜5案提示する** → ユーザーが件名を決定するのを待つ（**件名が決まるまで本文を書かない**）
3. **決定した件名に沿って本文を執筆**（**1,500字以上**必達。Discord URLは文字数に含まない）
4. **出力前に文字数チェック**: URL・署名を除いた本文が1,500字未満なら加筆してから出力する（1,500字未満での出力は不可）

> **理由**: どんな名文もクリックされなければ読まれない。件名は開封率を決める最重要要素。
> ユーザーが自分の件名案を持っている場合は、それを候補の1つとして尊重すること。

---

## 執筆前の必須確認（品質チェック）

`01_定義フォルダ/03_メルマガ作成ルール.md` の **「心が動く文章」品質基準** セクションを必ず読み、以下を守ること。

| チェック項目 | 基準 |
|---|---|
| 読後感 | 「この人おもろ！」「ついていきたい！」と思えるレベルか |
| CTA | 宣伝ブロックとして置かず、物語の帰結として自然に出現させる |
| 描写 | 時刻・場所・感情の具体名で「映像が浮かぶ」文章になっているか |
| エピソード | 架空の「Aさん vs Bさん」比較ではなく、実際の体験で語る |
| 感情動線 | トレース元を読んで感じた「グッと来る感覚」と同レベルか |

---

## 全型共通フォーマット

1. **件名**（20〜30文字。5パターン: 常識破壊型・好奇心フック型・問いかけ型・数字型・直球型）
2. **冒頭3行**（読み続けるかどうかが決まる。エピソード入り or 本題直球 or 問いかけ）
3. **ボディ**（各型の構成に従う。STEP2以上の情報深度・手触り感を必ず入れる）
4. **CTA**（1アクションに絞る。入れない回もOK）
5. **署名**
