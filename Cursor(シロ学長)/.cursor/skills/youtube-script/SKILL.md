# YouTube台本作成スキル

## ベーススキルの読み込み（最初に実行）

1. `shiro-master` スキルを読み込む → 「誰に・何のために・どう届けるか」の軸を確認
2. `goal-seek` スキルの「2. YouTube台本」セクションを確認 → ゴール3問（感情変化・行動・核心メッセージ）に答えてから進む

## 事前読み込み（必須 — 全て読み込んでから作業を開始すること）

### ① 定義・ルール
- `01_定義フォルダ/01_YouTube台本作成ルール.md`（役割・詳細ワークフロー）
- `01_定義フォルダ/04_ライティングスキル定義.md`（小学5年生翻訳・自己否定論破スキル）
- `02_テンプレート/YouTube台本テンプレート.md`（構成の型 ※この型に従って生成）

### ② アーカイブ（必須）

- **`03_アーカイブ/04_指導者FB/YoutubeFB/`（指導者フィードバック ― このフォルダ内の `.md` を毎回すべて読み込み、指摘を踏まえて執筆する。メルマガ用は sibling の `メルマガFB/`）**

## クイックスタート

ユーザーから台本作成の依頼が来たら、**以下の ヒアリング → Phase 0 → Step 1 → Step 2 の順番で実行すること**。

---

### 📋 事前ヒアリング（必須 — 毎回最初に実行）

台本作成の依頼を受けたら、構成案に入る前に必ず以下の3点を確認する。  
ユーザーが既に答えを提示している場合は確認済みとして次に進んでよい。

| # | 確認事項 | 質問例 |
|---|---|---|
| ① | **トレース元動画** | 「参考にしたい競合動画はありますか？URLまたは動画名を教えてください。なければ『なし』でOKです（Phase 0-Bのリサーチ結果から選びます）」 |
| ② | **フックの方向性・動画のニュアンス** | 「このサムネ・タイトルで一番伝えたいことは何ですか？視聴者にどんな感情・行動を起こさせたいか、イメージを教えてください」 |
| ③ | **差別化の核心テーマ** | 「今回の台本で一番の目玉にしたいポイントを教えてください（直近ニュース・価格根拠・手法・比較など）」 |

> **追加で聞いてよいこと：** サムネ画像・動画概要・ターゲットの特定セグメント（標準の30〜50代男性でよければ確認不要）

**ヒアリング完了後、ユーザーの追加返答を待たずに Phase 0 に自動で進むこと。**

---

ユーザーからテーマ・ターゲット・フック情報を受け取ったら、**以下の Phase 0 → Step 1 → Step 2 の順番で自動実行すること**（ユーザーから「Phase 0 は省略」と言われない限り必ず実行する）。

---

### 🔍 Phase 0：リサーチ（台本作成前に必ず実行・ユーザーの追加指示不要）

テーマが確定したら、台本作成に入る前に以下の **Phase 0-A → Phase 0-B の順番** でリサーチを実行し、インプットファイルを最新化する。

---

#### Phase 0-A：ニュース・時事リサーチ → `★ニュース・時事.md` に末尾追記

**目的：** 今回の台本で使えるインプット情報を最新化する。

**実行手順：**

1. 以下の観点で **Web検索を複数クエリ実行**する
   - 検索クエリ例（テーマが XRP の場合）：
     - 「XRP 最新ニュース 2026」
     - 「リップル 銀行 採用 2026」
     - 「XRP 価格 上昇 根拠」
     - 「XRP escrow 枯渇 最新」
   - テーマが変わった場合は対象コイン・トピックに合わせてクエリを変更する

2. 収集する情報の**優先順位**（上から順に重要）：
   - **① 数値データ**：市場規模・成長率・取引量・採用件数など、台本の根拠として使える具体的な数字
   - **② 成功事例・ケーススタディ**：実際に○○が採用した、△△の結果が出たなどの具体例
   - **③ 意外な事実**：「知らなかった」「え、そうなの？」と視聴者が感じる驚きのデータや情報

3. **鮮度の基準（厳守）**：
   - **最低でも直近6ヶ月以内**の情報に限定する（それより古い情報は除外）
   - 同じ条件なら**より直近の情報を優先**する

4. `07_インプット/YouTube台本用/★ニュース・時事.md` の**末尾に追記する**（既存の内容は絶対に削除しない）
   - 追記フォーマット：
     ```
     ---

     ## 【YYYY/MM/DD 追加｜[テーマ名]リサーチ】

     ### ニュース①：[見出し]
     - 日付：YYYY年MM月DD日
     - 内容：（3〜5文で要約）
     - 数値ハイライト：（台本で使える数字があれば抜き出す）
     - 台本活用メモ：（どのシーンで使えるか・どう使うか1文で）

     ### ニュース②：[見出し]
     ...（同様のフォーマットで続ける）
     ```

5. 追記保存後、チャットに「**ニュース・時事リサーチ完了：○件追加**」と報告する

---

#### Phase 0-B：競合台本リサーチ → `★競合台本.md` に末尾追記

**目的：** 今回の台本と被らないフック・構成を把握し、差別化ポイントを洗い出す。

**実行手順（Step B-1 → B-2 → B-3 の順番で必ず全て実行）：**

**Step B-1：ユーザー提供URLの優先処理（URLが提供されている場合）**
- ユーザーから特定のYouTube URLが提供されている場合、それらを最優先で処理する
- 各URLをWebFetchで取得し、タイトル・チャンネル・再生数・構成・トランスクリプトを確認する
- 今回の台本テーマと「関連性が高いもの」を選別して分析対象とする（関係ない動画は除外し、理由を記載）
- 拡散率（再生数÷チャンネル登録者数）が算出できる場合は算出する。できない場合は「算出不可」と記載
- **ユーザーが指定したリップル関連URLは拡散率2倍以上が基準。ビットコイン関連URLは拡散率5倍以上が基準**

**Step B-2：テーマ関連コインの追加Web検索（新規競合の発掘）**
- 今回のテーマ・動画タイトルに関連する**YouTube動画を新たにWeb検索**する（ユーザー提供URLとは別に実行）
- 検索クエリ例（テーマが XRP の場合）：
  - 「XRP 急騰 YouTube 2026」「リップル 仕込み 今すぐ」「XRP 価格予想 2026」など
- テーマが変わった場合は対象コイン・動画タイトルに合わせてクエリを変更する
- **採用基準（拡散率フィルター）**：
  - 拡散率 = 再生数 ÷ チャンネル登録者数
  - **主題コイン（XRP等）動画：拡散率2倍以上の動画のみ採用**
  - 拡散率が算出できない場合は、登録者数に対して明らかに再生数が多い動画を優先選択する

**Step B-3：今回のテーマコイン関連の追加検索（毎回必ず実行）**
- 今回の台本テーマに活用できそうな**テーマコイン（今回のサムネ・タイトルと同じコイン）の解説YouTube動画**を別途Web検索する
  - 例：今回の台本がXRP（リップル）なら → 「XRP リップル 急騰 YouTube 2026」「リップル 仕込み 今すぐ YouTube」など
  - 例：今回の台本がBTCなら → 「ビットコイン 急騰 YouTube 2026」「BTC 今すぐ買う理由 YouTube」など
  - テーマコインに合わせてクエリを変更すること
- **採用基準：拡散率2倍以上**（登録者1万人なら再生2万回以上）
  - 拡散率が算出できない場合は、登録者数に対して再生数が多いと判断できる動画を優先選択する
- 条件を満たす動画を2〜3本選んで分析し、Step B-1・B-2で採用済みの動画とは被らないようにする

**全Step共通：**
- `07_インプット/YouTube台本用/★競合台本.md` の**末尾に追記する**（既存の内容は絶対に削除しない）
- 追記フォーマット：
  ```
  ---

  ## 【YYYY/MM/DD 追加｜[テーマ名]競合リサーチ】

  ★1つ目
  [動画タイトル]
  [URL]
  チャンネル登録者数：約○万人 ／ 再生数：約○万回 ／ 拡散率：約○倍

  フック分析：（冒頭15秒の内容と「何が視聴者を引きつけているか」を3文で）
  構成分析：（大目次レベルで箇条書き）
  差別化メモ：（シロ学長の台本でこの動画と被らないようにすべき点）

  ★2つ目
  ...（同様のフォーマットで続ける）
  ```

- **ユーザー提供URLの全件について選別結果（採用/除外とその理由）を必ず一覧表形式で ★競合台本.md に記録すること**
- 追記保存後、チャットに「**競合台本リサーチ完了：○本追加**」と報告する

## 構成案フォーマット（Step 1）

大目次3つ × 各中目次3つ = 合計12ブロックで提示。  
出力後は「こちらの構成案でよろしいでしょうか？」とユーザーに確認する。

### 大目次タイトルのルール（Step 1 で必ず実施）
構成案を出力する際、大目次①②③のタイトルは以下の基準で生成すること：
- **視聴者が「え、そうなの？」と感じる逆説・意外性を含む**
- **ネガティブな問題提起（〇〇が全滅・〇〇は嘘）+ 解決策の予感がセットになる**
- 「〜だ」「〜する理由」「〜を全部バラします」のような断言・行動語を末尾に使う
- 日本語として自然に読めること（口語で違和感なく読めるか確認する）
- **【必須】3案をユーザーに提示し、ユーザーが選ぶ方式とする（自動選択してはならない）**

#### 大目次タイトル キーワード重複禁止ルール（厳守）
- 大目次①②③のタイトル間で同じ核心ワードを使い回さない  
  例：①で「3つの証拠」を使ったら、②では「3つの根拠」も類似表現のため禁止 → 「証拠」「根拠」「ポイント」が①②③に分散しないよう設計する
- 各大目次のタイトルで使うキーワードを下記に記録し、被りが出ないよう管理すること：
  - ①：___（数字・キーワード例：「93%」「現在地」）
  - ②：___（数字・キーワード例：「3つの根拠」「必然」）
  - ③：___（数字・キーワード例：「脱落」「分岐点」）
- 「オンチェーン」「ショートガンマ」「ガンマエクスポージャー」「資金調達率」「流動性」等の金融専門用語はタイトルおよび台本本文で使用禁止。必ず日常語（「プロが見るデータ」「お金を払って下落に賭ける」等）に置き換えること

#### 大目次③タイトル 固有ルール
- 大目次③の総括タイトルは、配下の中目次3-1のタイトルと被らない表現にすること  
  例：3-1が「老後2000万円問題をBTCで解決する3ステップ」なら、③のタイトルは「やることは3つだけ。でも9割が最後のステップで脱落します」のように俯瞰的表現にする
- 「1つの落とし穴」はタイトルに含めない（内容の中で機能させれば十分）

### 導入の書き方ルール（厳守）

---

#### 【導入の実践フロー＆テンプレート】（台本101・台本102の実績構造から抽出 — 必ずこの順番で書くこと）

導入は **フェーズA〜F の6段階** で構成する。各フェーズの文字数・行数を守ること。

---

**フェーズA：フック — 自己紹介前（5〜8行以内・絶対に超えない）**

- 目的：「この動画は自分に関係ある」「え、続き見たい」の2点だけを達成する
- 構成パターン：
  1. **断言1文**（サムネのキーテーマを含む。「可能性」「かもしれません」は禁止）
  2. **具体的事実を箇条書きまたは短い文で2〜4行**（数字・固有名詞を必ず入れる）
  3. **締めの1〜2文**（緊急性・損得感・自分ごと感）

- **〈実績例〉台本102型フック（計5行）：**
  ```
  リップルは3年後に取引所から消えます。

  ・在庫はピーク時から57%減
  ・アメリカは国の資産に指定
  ・日本の銀行も続々採用

  なのに、まだ1枚たった200円。
  今があなたの人生を変える最後のチャンスかもしれません。
  ```

- **〈実績例〉台本101型フック（計3行）：**
  ```
  楽天ユーザー4400万人のポイントが「投資資産」に？

  アメリカがリップルを「国家の財産」に指定し、
  ゴールドマン・サックスは暴落中に244億円を買い増し。

  あなたの楽天ポイント、このまま失効させますか？
  ```

- **フック執筆の禁止事項：**
  - ニュースの羅列で始めない（「楽天ウォレットがXRPを解放し〜」は禁止）
  - 「未来の日常」「意外な事実」「身近な損得」のいずれかの切り口で1文目を作ること
  - フックで8行を超えた場合は必ず削る

**【疑問提起の「マス層の不安」捉え方ルール】（台本104FBより追加）**

フェーズDの疑問提起は「今さら高値掴みになるんじゃないか」という投資経験者向けの不安だけでなく、以下のマス層（多数派）の本音を必ず捉えること：

- 「ビットコインって怪しくない？」「大損した人の話もよく聞くし……」（不信感・恐怖）
- 「難しそうで何から始めればいいか分からない」（知識ゼロの壁）
- 「仮想通貨=ギャンブル・投機」（偏見）

**対処法：** 「〜って思いますよね？8年前の私も全く同じことを思っていました」のように一人称転換型で共感→「でも〇〇が変わった」に展開すること。「今さら高値掴み」の不安は2つ目の疑問として追加するのはOK。

---

**フェーズB：自己紹介（3文固定）**

```
仮想通貨で年間10億円を運用するシロです。
2018年に500万円が200万円になった経験があります。
だからこそ、今日の話には自信があります。
```

- 「100以上のWeb3事業を手がける」等、視聴者に意味が伝わらない実績は禁止
- 「稼ぎながら」は禁止 → 「運用する」を使う

---

**フェーズC：テーマ提示（1文）**

「今日は〜をお話しします」の形で1文のみ。「〜について解説します！」は禁止 → 「〜をお話しします」に統一。

---

**フェーズD：疑問提起＋ミニ根拠（導入の「肉」。10〜20行が目安）**

> ここが最も重要。自己紹介の直後に何もなく3点予告に跳ぶのは禁止。「なぜこの動画を最後まで見るべきか」を10〜20行で視聴者に信じさせてから予告へ進む。

構成パターン：
1. **疑問提起（一人称転換型）** → 「〇〇と思いますよね。僕も最初はそう思ってました。でも〇〇で変わりました。」（禁止：「こんな風に思っていませんか？当然の反応です。」）
2. **ミニ根拠の展開（3点以内に圧縮）** → 3点予告の前に"超凝縮版"の根拠を先に見せることで「見る価値がある」と感じさせる
3. **「3つ目が一番ヤバい・大事」の予告煽り** → 最後まで見る動機を仕込む
4. **核心メッセージ（1〜2文）** → 動画全体のテーゼを1文に圧縮
5. **視聴ベネフィット（1文）** → 「今日の動画を最後まで見れば〇〇が全部分かります」
6. **「あなたはまだ持ってないのに。」フレーミング（任意）** → ニュース列挙の締めに使うと行動喚起に直結する

- **〈実績例〉台本102型フェーズD：**
  ```
  「いやいや、リップル100万円は盛りすぎでしょ」
  って思いますよね？私も最初はそう思ってました。

  でも、調べれば調べるほど
  「上がらない方がおかしくない？」に変わりました。

  理由は3つ。

  1つ目、世界中の銀行がリップルを「使い始めた」。
  これは予測ではなく、現実です。

  2つ目、取引所の在庫が57%消えた。最速3年で底をつきます。

  そして3つ目、これが一番ヤバいんですが──
  リップルは「値上がりしないとシステムが壊れる」構造になってます。

  つまりリップルは、人気だから上がるのではなく、
  世界の送金システムが止まるから上がるんです。
  ```

#### 【冒頭15秒フックの最適化ルール】（厳守）

冒頭15秒（目安：衝撃の結論〜動画テーマ提示まで）は**視聴者が得られる最大のポジティブなメリット**を前面に出すこと。

- 「〜が分かる」「〜できるようになる」など**得られる未来像**を明確に言語化する
- 恐怖・不安の煽りだけで終わらず、必ず「この動画を見れば〇〇になれる」という**希望のベクトル**をセットにする
- 例：「今日この15分を見れば、あなたは資産を3倍にするための具体的なロードマップを手に入れられます」
- 衝撃の結論はネガティブな問題提起でもよいが、**直後の1文で必ずポジティブな出口を示す**こと

> 冒頭離脱は致命的。最初の15秒で「見続ける理由（メリット）」を与えることを最優先にすること。

**絶対禁止事項（導入での NG）**
- 衝撃の結論を3行以上書かない
- 自己紹介の後に不自然な接続文（「少し聞いてほしいのですが」等）を入れない
- 視聴者への肯定・褒め（「あなたは上位数%です」「偉いです」等）は導入では使用しない → 本編・大目次①で使うこと
- 日常例（おにぎり・NISA等）を問いかけなしに唐突に出さない
- 冒頭15秒をネガティブな問題提起だけで終わらせない（必ずポジティブな出口とセットにする）
- 話題の切り替え（NISAの話・仮想通貨の話など）は前の話の結論を1〜2文でまとめてからブリッジすること。結論なしに「ということで今回は〜」と繋ぐのは禁止（けいたFB反映）
- 「インサイダー級」「黎明期」「一長一短」などの業界用語・難解表現を使わない → 完全に削除するか小学5年生レベルの言葉に置き換えること（けいたFB反映）
- **「歴史的瞬間」「最初で最後のチャンス」などの抽象的な煽り表現は禁止**（FB101反映）→ 新規視聴者に「またこの手の動画か」と思われる。代わりに「楽天ポイント」「老後資金」など身近・具体的な切り口から入ること
- **「突然ですが」は冒頭に使わない**（FB101反映）→ テンポが落ちる上に視聴者への「引き」がゼロ
- **「15分だけお付き合いください」は禁止**（FB101反映）→ 尺を意識させて逆効果。「ここだけでも見てください」のように内容の魅力で引っ張ること
- **「ただのポジショントークでしょ？」は自分で信用を下げるため禁止**（FB101反映）→ 視聴者がまだ思っていないネガティブワードを植え付けてしまう。代わりに「大損した人の話も聞くし…」など視聴者が実際に持っている不安に寄り添うこと
- **「サムネにもあった」は完全禁止**（FB101反映）→ 動画内でサムネに言及すると視聴者の意識が動画外に向く
- **「〜について解説します！」は過信的な上から目線ニュアンスがあるため「〜をお話しします」「〜を見ていきましょう」に変更**（FB101反映）
- **「少し話は変わりますが」は離脱トリガーになるため禁止**（FB101反映）→ 話が逸れると思った瞬間に視聴者は離れる。代わりに「先日〜で衝撃的な光景を見ました」のようなフックで繋ぐ
- **「素晴らしいです」（視聴者へのコメント）は上から目線に聞こえるため禁止**（FB101反映）
- **「99%の人が〜と思ったはず」のような根拠のない断言は禁止**（FB101反映）→ 視聴者を見下している印象を与えるリスクがある。「〜と思いましたよね」に変更すること
- **「めちゃくちゃ危険です」などの過剰な煽り表現は禁止**（FB101反映）→ 新規層が引く。論理で引き込むこと（例：「楽天は最初のドミノに過ぎません。問題はあなたの老後のお金が足りるかどうかです」）
- **「世界最先端」「プレミアム」などの装飾過多な形容詞は禁止**（FB101反映）→ YouTube視聴者は「盛りすぎ」の表現に敏感で逆に信用を下げる

- **「勿論」は使用禁止**（FB102反映）→ ひらがなで「もちろん」を使用すること

### 【FB102指導者FBから得た重要ルール集】（FB102反映・必須）

台本を書く際、以下のルールを全て守ること。これらは指導者からのFBを元に抽象化したものであり、次回以降同じ指摘を受けないための必須チェックリスト。

#### A. 冒頭・構成関連

1. **冒頭1行目は断言が鉄則** → 「可能性が出てきました」「〜かもしれません」は弱すぎる。「○○します。」「断言します。」で始める
2. **目次で「3つ目が一番大事・絶対最後まで見てください」と先に予告する** → 「3つ目に失敗談を絡める」ことで最後まで見る動機を作る。「順番に全部証拠付きで見せます」で根拠ある動画だというブランドを設定する
3. **文脈と合わないベネフィット提示は禁止** → 「リップルの話をしているのに突然『老後2000万円問題が解決できます』」のように文脈が飛ぶと信頼感が下がる。代わりに「この動画を最後まで見れば○○の根拠が全て分かります」という動画視聴の直接的なメリットを提示する
4. **前半CTA（取引所紹介等）は4行以内に収める** → 「シロ学長愛用の」「フェラーリともスポンサー契約を結んでいる」等の営業ワードは禁止。「僕が使っているのはこの取引所です」でシンプルに。「はい、では本題に戻ります」で切り替えを明示する。**ここが長いと視聴維持率が一気に下がる**

#### B. 説明パート（本編）のテンポ維持

5. **比喩は「より強い印象を与えるもの」を選ぶ** → 古さ・原始さを表現するなら「郵便屋さん」より「伝書鳩」。「一定期間ごとに自動的に開く金庫」より「タイマー付きの金庫」。「〇〇年なのに△△」という時代との対比で納得感を先に仕込む
6. **説明パートに1〜3分（約600字）に1回フックを必ず挿入する** → 「でもここがヤバいんですが──」「〇〇って聞いたことありますか？」「〇〇じゃないですか？」等。**長尺動画の視聴維持率は「小さなフックの連続」で決まる**
7. **専門用語の前に「○○って聞いたことありますか？」で心理ハードルを下げる** → 問いかけてから説明に入ると視聴者の受け入れ態勢が整う
8. **「あなたはまだ○○してないのに。」フレーミングを活用する** → ニュース列挙の締めに「国も銀行も投資家も全員確保しに来てる。あなたはまだ持ってないのに。」と追加することで、乗り遅れ感と行動喚起を同時に演出できる
9. **シミュレーションにリスク提示＋次パートへの橋渡しをセットにする** → 「もちろん、一直線にここまで行くわけじゃない。途中で50%下がることも普通にある。でもその暴落が『チャンス』になる方法を、この後お話しします。」という形で次セクションへ自然に繋げる

#### C. 失敗談・感情パートの設計

10. **失敗談はパンチラインを先出しする** → 「なぜかというと過去の私がそうだったから」という順序ではなく、「そして300万円消えました」という具体的な金額・結果を先出しして「なぜ？」という好奇心で次のセクションへ引っ張る。**「かなり危険」「可能性があります」等の弱い表現より、具体的な金額で危険性を示す方が100倍伝わる**
11. **「誰も教えてくれない」等の希少性ワードを修飾に使う** → 「『買って放置』には、誰も教えてくれない3つの落とし穴があります。」のように、視聴者が「ここでしか聞けない」と感じる表現を使う

#### D. CTA・エンディング

12. **特典は最強の2つを「有料級」と先出しして「全部で○つ」とまとめる** → 5つ全部を羅列すると離脱の原因になる。「特に○○と○○は正直有料級なので、受け取らないと損です。全部で5つあります。」の構成にする
13. **「登録しないでください」の逆説フィルタリングを使う** → 「ただし『何も勉強したくない』『ツールで自動的に稼ぎたい』って人は登録しないでください。」という逆説表現で「本気の人だけ来てほしい」スタンスを示し、結果的に登録率が上がる
14. **視聴者の気持ちをカッコ書きで代弁するのはNG** → 「○○と思ってくれた方は〜」のような代弁は押し付けがましくなる。「今日の内容をもっと深く知りたい人は〜」のように直接的に語りかける
15. **エンディングのコメント促進は具体的なYes/No質問にする** → 「今日の学びをアウトプットしましょう」は抽象的すぎてコメントが書きにくい。「○○ですか？○○じゃないですか？」という二択Yes/No質問にして、「〇〇じゃない人は『まだ』って書いてください」のアンカリングで行動前提を植え付ける

### 大目次間の「橋渡し」ルール（必須）
各大目次の冒頭は、必ず前の大目次の**結論フェーズ**を踏まえた繋ぎ言葉から始めること。

**設計の手順：**
1. 前の大目次が終わった時点で「視聴者はどのフェーズにいるか」を1行で言語化する
2. その視聴者フェーズから**自然に湧いてくる次の疑問**を言語化する
3. 大目次の冒頭で「その疑問への答えを今から話す」と宣言してから本文に入る

**フェーズ遷移の例（量子BTC台本の場合）：**
- 大目次①終了後：「量子でBTCが終わるわけじゃないことは分かった」
- 大目次②への疑問：「でも安全なだけで上がるとは限らない。本当に上がるの？」
- 大目次②冒頭の宣言：「今から、上がる理由を2つのデータで証明します」

**絶対禁止：**
- 前の大目次と内容が断絶した状態でいきなり新テーマに入ること
- 「次に〇〇について話します」だけで繋ぐこと（視聴者フェーズの受け取りがない）

### 中目次間の「橋渡し」ルール（必須）

大目次内の各中目次（例：1-1 → 1-2 → 1-3）の繋ぎも同様に設計すること。

**設計の手順：**
1. 今の中目次を読み終えた視聴者が「次に自然に湧く疑問」を1行で言語化する
2. 中目次の末尾でその疑問を「先取り予告」する（「次で〜を解説します」ではなく「〜な疑問が出るはずです。それが次のポイントに直結します」のように予感を持たせる）
3. 次の中目次の冒頭は「その疑問への答え」として自然に始める

**禁止：**
- 「次に〇〇について話します」だけで繋ぐこと（疑問の先取りなし）
- 前の中目次の内容と断絶した状態で突然新トピックに入ること
- 各中目次が「単独で完結」しており、前後と論理的に繋がっていない構成

動画の中盤（大目次①終了〜大目次②のどこか1箇所）に、必ず**後半への予告文**を1〜2文挿入すること。

**目的：** 中盤で離脱しようとしている視聴者に「最後まで見る理由」を与える。

**フォーマット例（そのまま使ってよい）：**

**ルール：**

### 疑問提起の設計ルール（必須）

台本内の「〇〇ですよね？」「〇〇だと思いますよね？」という疑問提起は、以下の基準で設計すること。

**原則：「ここまでの動画を見てきた仮想通貨初心者が、今この瞬間頭に浮かべている疑問」だけを使う**

1. **視聴者フェーズに合わせる：** まだ動画内で説明していない前提知識を必要とする疑問は使わない。「この動画のここまでを見た視聴者」が実際にその疑問を持つかを必ず確認する。
2. **初心者基準で設計する：** 仮想通貨に詳しくない30〜50代男性が「そうそう、それが気になってた」と感じる疑問にする。業界知識を前提とした疑問は禁止。
3. **ライター都合の疑問を禁止する：** ストーリーを進めるためだけに作った疑問（視聴者が実際には思わない疑問）は禁止。
4. **【未経験者2：経験者1の割合ルール】（けいたFB反映）：** 仮想通貨の疑問定義を列挙する際は「まだ買ったことがない人の不安・疑問」を2つ、「過去に買って失敗した経験者のマイナス体験」を1つの割合で構成すること。視聴者の大半は未経験者のため、経験者向けの疑問ばかりを並べると刺さらない。
5. **【複数の感情を表現する】（けいたFB反映）：** 「怪しい」という一方向の感情だけでなく「怖い」「難しそう」「NISAでいいでしょ」「今さら遅い」など複数の感情パターンを並べること。目的は「自分ごと」と感じさせることなので、感情のバリエーションを出す。

**検証方法（疑問提起を書くたびに確認）：**
「仮想通貨を全く知らない40代男性が、この動画のここまでを見た状態で、本当にこの疑問を持つか？」→ YESなら使用可、NOなら書き換える。

- 各特典に「—— だから価値がある」という**ベネフィットの一言**を必ず追加すること（ただのリスト羅列は禁止）
- **「世界最先端」「プレミアム」等の装飾過多な形容詞は禁止**（視聴者に「盛りすぎ」と思われて信用を下げる）
- パーソナルな要素（「私が実際に毎日使っているものだけ厳選」等）を入れて希少性を出す

**良い例：**
- 「1つ目。トレードロジックの完全解説動画—— これだけでもメルマガ登録の価値があります」
- 「3つ目。Web3の必須ツール・サイト集—— 私が実際に毎日使っているものだけ厳選」
- 「5つ目。ライフプランシート—— 埋めるだけで『あと何年で2000万円に届くか』が見える」

### 大目次③の構成ロジック（必須・順番厳守）

#### 伝えるべき主メッセージ（毎回固定）
大目次③は毎回以下の3段論法で構成すること（見せ方・訴求角度は毎回変える）：
1. **ガチホの落とし穴を見せる**（「買って放置すればいい」という視聴者の思い込みを崩す）
2. **トレードはガチホの敵ではなく相棒だと伝える**（元本を育てる＋相場を読める = ガチホの力が最大化する）
3. **怖い→原因を知れば怖くない→解決策→CTA**（「トレードが怖い」の感情を論理で解除し、正しい環境への行動を促す）

#### ①〜③への展開（固定フロー・台本102版）
1. **【ガチホの3つの落とし穴】**（実際の暴落チャートを見せながら具体的に演出する）
   - ①パニック売りリスク（95%暴落チャート：500万→25万の恐怖）
   - ②買い増しチャンスを逃す（含み損で動けない状態の損失）
   - ③時間という最大の資産を失う（3年間のキャリー損：知人との差）
2. **【トレードはガチホの相棒】**（「ガチホか？トレードか？」の二択ではなく「両方が最強」を伝える）
   - 元本が育つ（トレード利益でリップル買い増し→雪だるま式）
   - 相場が読めるようになる（暴落をパニックではなくチャンスと判断できる）
3. **【怖い？→原因を知れば怖くない】**（感情の解除→論理的裏付け→解決策→CTA）
   - 「怖いのは負ける理由を知らないから。知れば怖くない」
   - **敗因①：自己流＝再現性ゼロ**（野球フォームの例え。毎回違うフォームでは永遠に上手くならない）
   - **敗因②：ツール・他人任せ**（相場を読む力がつかない。ツールが通用しなくなった瞬間に崩壊）
   - **自己開示**（「私もこの2つどちらもやって大損した」→共感・信頼）
   - **解決策**（再現性のあるロジック＋AIインジケーター（補助輪）で自分の判断力を育てる）
   - **成果報告**（直近の台本に掲載していない新しい声を `episode.md` から引用）
   - **海外エピソード**（マニラ等の「日本が遅れているだけ」感を演出し緊迫感を与える）
   - **覚悟のCTA**（「勉強したくない・ツールで自動に稼ぎたい人はご遠慮ください。本気の人だけ登録」）

> **絶対禁止：**「あなたは正しかった」から始まる共感文は使わない（FB102反映）。自己開示型（「私も過去に〜」）で代替すること。

### 判定基準

| 文字数 | 対応 |
|---|---|
| 10,000文字未満 | **加筆必須**。不足分を大目次・中目次に追記してから再計測する |
