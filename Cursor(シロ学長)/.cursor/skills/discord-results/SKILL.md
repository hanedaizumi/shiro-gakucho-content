# 成果報告作成スキル（Discord / YouTube / X 個別出力）

## 事前読み込み（必須）

- `05_Discord運用/プロンプト/成果報告プロンプト.md`（4種類のプロンプト全文）
- `.cursor/rules/youtube-x-plain-format.mdc`（一般型のYouTube/X・Discord本文の標準言い回し）

## 入力情報の確認

ユーザーから以下の2点を受け取る:

1. **タイプの指定**: 「一般型」または「インジケーター訴求型」
   - **【タイプは毎回ユーザーが指定する】（2026/6/20〜・絶対厳守）**: タイプの指定がない場合は作業を開始せず、必ず「一般型・インジケーター訴求型のどちらですか？」と確認する。デフォルトで自動判断・自動選択しない。
   - **一般型**：シロ学長の市況・トレードロジック・コミュニティ環境が成果の理由。インジケーター・ツールの機能説明は一切出さない
   - **インジケーター訴求型**：ツールの機能・改善点を前面に出した成果報告
2. **成果画像**: 受講生の成果画像（1枚〜複数枚）をアップロードしてもらう

### 画像からの情報読み取り（優先）
- アップロードされた画像から「成果金額」「画像内のキャッチコピー」「【学び】コメント」を自動読み取りする
- テキストでの詳細入力は不要。画像さえあれば成果報告を作成できる
- 画像が複数ある場合はすべての成果を集約して1つの報告にまとめる

タイプが不明な場合のみ確認してから作業を開始する。

## 文章量ルール（2026/5/20〜・必須・YouTube/Discord両方に適用）

- **受講生コメント（成果行）**：1件あたり**1行・25文字以内**に圧縮する。長い【学び】コメントは要点だけを残して短縮する。
  - 例）「シロ学長の特別市況とインジケーターに従って大きく勝てました。初心者でもシロ学長のお陰でプラスが続いていて本当に有り難いです」→「市況＋インジケーターに従うだけで17万超え！初心者でもプラスが続いてます」
- **本文（核心・CTA前）**：**6行以内**に収める。借金エピソード・ツールの必然性・行動促進の3点を各1〜2行で端的に書く。冗長な接続詞・繰り返しは削る。
- YouTube/X用・Discord用の**両方**に同じ圧縮基準を適用する。

## 装飾ルール（2026/4/24更新・全タイプ共通）

- **YouTube / X 用（コードブロック内）**: 見出し用 `#` ・太字用の `*`（`**` 含む）を**一切使わない**。タイトル・中見出しは1行プレーンテキスト、成果行は `💰+金額「コメント」` または `+金額「コメント」`（先頭に余計な `-` なし）でよい
- **Discord 用（コードブロック内）**: `#` / `##` / `**` など**マークダウン装飾を使う**（従来どおり）

### before→after→実際の声（一般型・共通ルール）
- 複数の成果金額を紹介した後、**画像の中で最も「変化の落差が大きい人」または「共感しやすいストーリーを持つ人」を1名選ぶ**
- before: その人の以前の状態（悩み・失敗・不安）を1〜2行で
- after: 実践後どう変わったか（結果）を1行で
- 実際の声: 画像内の【学び】コメントをそのまま（または25字以内に圧縮）引用
- 煽りすぎない。「ありえない！」「狂った！」等の過剰な感嘆は使わない

1. **足確定後のみ点灯・消えない**：後出し消滅を完全撤廃

4. **画面シンプル化**：LONG / SHORT / ✖ の3表示のみ。余計なライン非表示
