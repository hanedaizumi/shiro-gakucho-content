# YouTube台本フィードバック スキル（テクニカルチャンネル専用）

## このスキルの目的

テクニカルチャンネルの本質は「**シロ学長の信者（ファン）を作るビジネス**」。
単に正しいことを言うだけでなく、**「この人の分析は本物だ」「この人の言うことをもっと聞きたい」** と視聴者が感じる台本になっているかを診断する。

---

## フィードバックの3大観点

台本を評価する際、以下の3軸を必ず診断すること。

---

### 観点①：わかりやすさ（Clarity）

**目標：** 「ロジックがめちゃくちゃ理解できた」と視聴者に感じさせる

視聴者は「分析の答え（上がる/下がる）」だけでなく、**「なぜそう判断するのか」**のロジックを追えるかを見ている。

**チェック項目：**

| # | チェック内容 | OK / NG |
|---|---|---|
| 1 | 「前提 → 根拠 → 結論」の3段構成になっているか | |
| 2 | 専門用語の直後に日常の例えが入っているか | |
| 3 | 「なぜ」がすっ飛ばされていないか（断言の後に理由がセットになっているか） | |
| 4 | チャートのどこを見ているかが言語化されているか（「この斜めライン」だけでなく「2年前から効いてる〇〇のライン」など） | |
| 5 | 1文で2つ以上のことを言っていないか（1文1概念） | |
| 6 | 条件が具体的な数字で示されているか（「下がったら」ではなく「7万5,500ドルを実体で割ったら」） | |

**わかりやすさのNG例 → OK例：**
```
NG: 「ここのラインを割れると下がります」
OK: 「4時間足の終値ベースで7万5,500ドルを割り込んだ場合、次の下値メドは7万2,000ドルです。ヒゲは無視、実体の確定を待つのがポイント」
```

---

### 観点②：権威性（Authority）

**目標：** 「この人の分析は本物だ」「他のYouTuberとは違う」と視聴者に感じさせる

権威性は「自慢」ではなく「実績に根拠を乗せること」で生まれる。

**チェック項目：**

| # | チェック内容 | OK / NG |
|---|---|---|
| 1 | 過去の予測が的中した実績への言及があるか（「前回お話しした〇〇ドルラインで反発」等） | |
| 2 | 分析の根拠が2段以上重なっているか（単一根拠で断言していないか） | |
| 3 | エントリーポイント・損切りライン・利確目標が具体的な数字で示されているか | |
| 4 | シロ学長自身のトレード実績・失敗談が適切に使われているか（捏造禁止・`00_チャンネル設計前提.md`の実績のみ） | |
| 5 | 他の発信者との差別化が暗示または明示されているか | |
| 6 | 「個人の意見」で終わらず、チャートの構造・データが根拠になっているか | |

**権威性の参考スタイル（ベンチマーク）：**

- 村上さん型：「2023年3月からビットコインが8万8千ドルを超えると一貫して言い続けた。結果、その通りになった」→ **長期での一貫性**で権威を示す
- トレーダーRed型：「昨日の動画でお伝えしていた7万4,000〜7万5,000ドル付近で反発、矢印方向に2,500ドル幅上昇した」→ **直近予測の的中**で権威を示す

シロ学長テクニカルは両方を組み合わせること。

---

### 観点③：ファン化（Fan-building through Information）

**目標：** 「この人、俺の知らないこと知ってる！」と感じる瞬間を1箇所以上作る

**最重要原則：**
> ファン化は「情報」で行う。感情的な煽りや過剰な自慢でやらない。
> 必要ない場面（挨拶・CTA・まとめなど）でファン化要素を無理に差し込まない。
> 情報の「質と深さ」そのものがファンを作る。

**チェック項目：**

| # | チェック内容 | OK / NG |
|---|---|---|
| 1 | 「知らなかった」「そういうことか！」と思える**新しい視点・知識**が1箇所以上あるか | |
| 2 | その「気づき」は情報（データ・ロジック・チャート構造）で届けられているか（感情煽りでないか） | |
| 3 | 視聴者の「当たり前の認識」を一度裏切る逆説の事実が含まれているか | |
| 4 | 「教える感じ」ではなく「少し先を歩く先輩が教えてくれる感じ」になっているか | |
| 5 | ファン化要素が台本全体の20〜30%以内に収まっているか（多すぎると情報の薄い動画になる） | |

**ファン化のOK例（情報ベース）：**

```
「出来高が急増してるのに価格が下がってる。
 これ、99%の人は『なぜか下がってる』で終わる。
 でも出来高を見てる1%の人間には、機関が売り抜けてる
 サインだとすぐ分かる。今のBTCがまさにその形。」
```

→ 「出来高 = 機関の動き」という新しい視点を提供し、「知ってれば防げた」という悔しさと同時に「この人はそれを知っている」という信頼感を生む。

**ファン化のNG例（煽りベース）：**
```
NG: 「このチャンネルを見てるあなたは上位1%です！」
NG: 「他の発信者は嘘をついています！」
NG: まとめパートで突然「そんなシロ学長のDiscordに…」と始まる
```

## 台本の粒度・情報の抽象度 基準

以下の2タイプを参考に、テクニカルチャンネルとして適切な粒度か確認すること。

### Type A：村上さんスタイル（実績語り型）
- 長期予測の的中 → 実績で権威を示す
- 「寝れない夜が続いた」等の感情的な重さで視聴者と共感
- 結論先出し（8万8千で全利確）→ 理由を後から説明
- 具体的なポジション管理の話（複数ポジション・追加購入の水準）

→ シロ学長テクニカルでは：「過去の分析との整合性」と「自身のトレード経験」をセットで語る場面で活用

### Type B：トレーダーRedスタイル（純テクニカル型）
- 毎回昨日・直近の予測的中を確認してから本題へ
- シナリオA/B/Cを条件付きで提示（「ここを割れたら〜」「ここを抜けたら〜」）
- 損切りライン・利確目標を毎回具体的な数字で提示
- 15分以内でコンパクトにまとめ

→ シロ学長テクニカルでは：本編の分析パート（シグナル解説・アクションプラン）で活用

### 適切な粒度の基準
- 数字：「下がる」ではなく「7万5,500ドルを実体で割ったら」
- 条件：「上がったらロング」ではなく「陽線確定＋出来高が20日平均の1.5倍以上で短期ロング候補」
- 時間軸：「週足で確認」「4時間足の実体ベース」を明示
- 具体的なアクション：「どう動くか」まで落とし込む

## NG表現リスト（テクニカルチャンネル専用）

| NG表現 | 理由 | 置換先 |
|---|---|---|
| 「誰でも」「簡単に」 | リテラシー高い層が離れる | 「条件が揃ったら」「根拠を確認してから」 |
| 「絶対に上がります」 | 根拠なき断言 | 「このシナリオでは〜が最有力」 |
| 「今が最後のチャンス」 | 根拠なき煽り | 「このラインを割れると次の買い場は〇〇まで下落する可能性がある」 |
| 「俺だけが知っている」 | 反感を招く | 情報の希少性は「データ」で示す |
| 「スマホで5分で稼げる」 | チャンネルコンセプト違反 | 禁止（削除） |
| 「チャンネル登録お願いします」を本編中に連発 | テンポを殺す | エンディングのみ1回 |

## 4施策チェック（5本目以降の台本に必須）

`technical-channel-principles.mdc` に定める4施策が台本に反映されているか確認する。

| 施策 | チェック内容 |
|------|-----------|
| **①権威性強化** | 冒頭自己紹介にエピソードつき権威性アピールが入っているか（「〇年前に数千万溶かした→今は月利〇〇万」の形式） |
| **①権威性強化** | エピソードが直近2〜3本の台本と被っていないか |
| **②Vantage誘導** | 中盤CTAにVantageの紹介が1回だけあるか |
| **②Vantage誘導** | 末尾にVantageの単独誘導がなく、「Discord内で手順を確認」の導線になっているか |
| **③コメント誘導** | まとめの前にYes/No型コメント誘導が入っているか |
| **③コメント誘導** | 「全部読んでます」のフレーズが入っているか |
| **④次回予告** | 後半CTAに「次回は〜」の予告が入っているか |
| **④次回予告** | 具体的な価格帯・条件に言及しているか |

→ 4施策いずれかが欠けている場合は「施策①〜④不足」として優先対応TOP3に必ず含める。
